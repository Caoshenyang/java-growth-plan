# 分布式电商系统

> 综合实战项目：全栈电商系统

---

## 🎯 项目目标

构建一个完整的分布式电商系统，综合运用所学技术：

- [ ] 微服务架构
- [ ] 服务治理
- [ ] 分布式事务
- [ ] 搜索引擎
- [ ] 消息队列
- [ ] 缓存优化
- [ ] 高可用设计

---

## 📋 系统模块

### 核心服务

| 服务 | 功能 | 技术栈 | 状态 |
|------|------|--------|------|
| 用户服务 | 注册、登录、个人信息 | Spring + MySQL + Redis | 📋 |
| 商品服务 | 商品管理、分类、搜索 | Spring + ES + Redis | 📋 |
| 库存服务 | 库存管理、扣减、回滚 | Spring + MySQL + Redis | 📋 |
| 订单服务 | 订单创建、查询、状态流转 | Spring + MySQL + MQ | 📋 |
| 支付服务 | 支付对接、回调、退款 | Spring + MQ + MySQL | 📋 |
| 营销服务 | 优惠券、活动、秒杀 | Spring + Redis | 📋 |
| 物流服务 | 物流查询、轨迹同步 | Spring + MySQL | 📋 |

### 支撑服务

| 服务 | 功能 | 技术栈 | 状态 |
|------|------|--------|------|
| 网关服务 | 路由、限流、鉴权 | Spring Cloud Gateway | 📋 |
| 注册中心 | 服务注册与发现 | Nacos | 📋 |
| 配置中心 | 配置管理 | Nacos | 📋 |
| 链路追踪 | 调用链路追踪 | SkyWalking | 📋 |

## 💡 核心流程

### 下单流程

```
1. 用户点击"立即购买"
   │
2. 校验商品状态 (商品服务)
   │
3. 校验库存 (库存服务 - 预扣)
   │
4. 计算价格 (营销服务 - 优惠券)
   │
5. 创建订单 (订单服务)
   │
6. 发起支付 (支付服务)
   │
7. 支付成功回调
   │
8. 扣减库存 (库存服务 - 确认)
   │
9. 更新订单状态 (订单服务)
   │
10. 发送消息 (MQ - 物流、消息通知)
```

### 分布式事务处理

```java
@GlobalTransactional
public void placeOrder(Order order) {
    // 1. 扣减库存
    stockService.deduct(order.getSkuId(), order.getQuantity());

    // 2. 创建订单
    orderService.create(order);

    // 3. 使用优惠券
    couponService.use(order.getCouponId());

    // 自动回滚处理
}
```

## 🚀 技术栈

### 后端

| 类别 | 技术 |
