## 性能报告模板

```markdown
# [项目名称]性能测试报告

## 1. 测试概述

### 1.1 测试目标
- QPS目标：100,000
- 响应时间P99 < 100ms
- 错误率 < 0.1%

### 1.2 测试环境
| 组件 | 配置 |
|------|------|
| CPU | 8核 |
| 内存 | 16G |
| 网络 | 千兆 |

### 1.3 测试工具
- JMeter 5.5
- K6
- wrk

## 2. 测试场景

### 2.1 场景1：正常负载
- 并发用户：100
- 持续时间：10分钟

### 2.2 场景2：高峰负载
- 并发用户：1000
- 持续时间：30分钟

### 2.3 场景3：极限压测
- 并发用户：10000
- 持续时间：5分钟

## 3. 性能指标

### 3.1 QPS
| 场景 | 目标QPS | 实际QPS | 达标 |
|------|---------|---------|------|
| 正常 | 1,000 | 1,200 | ✅ |
| 高峰 | 10,000 | 8,000 | ❌ |
| 极限 | 100,000 | 50,000 | ❌ |

### 3.2 响应时间
| 指标 | 目标 | 实际 | 达标 |
|------|------|------|------|
| P50 | < 50ms | 30ms | ✅ |
| P99 | < 100ms | 150ms | ❌ |
| P999 | < 500ms | 800ms | ❌ |

### 3.3 错误率
- 正常：0.01% ✅
- 高峰：0.5% ❌
- 极限：5% ❌

### 3.4 资源使用率
- CPU：60%
- 内存：70%
- 网络：40%

## 4. 瓶颈分析

### 4.1 主要瓶颈
1. 数据库连接池不足
2. 缓存命中率低
3. 线程池配置不合理

### 4.2 问题定位
```
# 慢查询
#1 SELECT * FROM user WHERE name LIKE '%张%'
耗时：2s

# 错误日志
java.sql.SQLException: Connection pool exhausted
```

## 5. 优化方案

### 5.1 数据库优化
- 增加连接池大小：20 → 50
- 优化慢查询：添加索引
- SQL优化：避免LIKE '%...%'

### 5.2 缓存优化
- 启用Redis缓存
- 缓存热点数据
- 预热缓存

### 5.3 应用优化
- 调整线程池参数
- 使用异步处理
- 连接池优化

## 6. 优化效果

### 优化前后对比
| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| QPS | 50,000 | 120,000 | 140% |
| P99 | 150ms | 80ms | 47% |
| 错误率 | 5% | 0.05% | 99% |

## 7. 结论与建议

### 结论
经过优化，系统性能大幅提升，达到预期目标。

### 建议
1. 继续监控性能指标
2. 定期进行压测
3. 建立性能基线
4. 制定容量规划
```

## 性能指标定义

### QPS（Queries Per Second）
- 每秒查询数
- 衡量系统吞吐量

### TPS（Transactions Per Second）
- 每秒事务数
- 衡量系统处理能力

### 响应时间
- P50：50%请求的响应时间
- P95：95%请求的响应时间
- P99：99%请求的响应时间
- P999：99.9%请求的响应时间

### 错误率
- HTTP错误码占比
- 业务异常占比

### 并发数
- 同时在线用户数
- 同时处理请求数

## 工具使用

### JMeter
```bash
# 启动JMeter
jmeter

# 命令行运行
jmeter -n -t test.jmx -l result.jtl -e -o report

# 查看报告
open report/index.html
```

### K6
```bash
# 运行测试
k6 run --out json=result.json script.js

# 生成报告
k6-reporter result.json --output out.html
```

### wrk
```bash
# 压测
wrk -t12 -c400 -d30s --latency http://example.com

# 参数说明
-t: 线程数
-c: 连接数
-d: 测试时长
--latency: 显示延迟统计
```
